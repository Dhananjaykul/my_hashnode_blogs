### **ğŸ”¹ Step 2: Configure Git**

Set up your username and email:

```bash

git config --global user.email "your.email@example.com"
```

To confirm the configuration:

```bash
git config --list
```

This stores your Git identity for tracking changes.

## **2ï¸âƒ£ Initializing a Git Repository**

### **ğŸ”¹ Step 3: Create a Project Folder & Initialize Git**

## **3ï¸âƒ£ Creating & Tracking Files**

### **ğŸ”¹ Step 4: Create a File & Check Git Status**

```bash
touch index.html
echo "<h1>Welcome to Git</h1>" > index.html
git status
```

This will show that `index.html` is an **untracked file**.

### **ğŸ”¹ Step 5: Add File to Staging Area**

```bash
git add index.html
```

Alternatively, add all modified files:

```bash
git add .
```

### **ğŸ”¹ Step 6: Commit Changes**

```bash
git commit -m "Initial commit - Added index.html"
```

**What does** `commit` do?  
A commit captures a snapshot of your changes in Git. Run the following to check commit history:

```bash
git log --oneline
```

## **4ï¸âƒ£ Making Changes & Tracking Versions**

### **ğŸ”¹ Step 7: Modify the File & Check Changes**

Modify `index.html`:

```bash

```

Check the status:


### **ğŸ”¹ Step 8: Commit New Changes**

```bash

```

Now check commit history again:

```bash

```

## **5ï¸âƒ£ Reverting to Previous Versions**

### **ğŸ”¹ Step 9: View an Older Version of the File**

List commits:

```bash
git log --oneline
```

Copy any commit ID and run:

```bash
git show <commit-id>:index.html
```

### **ğŸ”¹ Step 10: Reset a File to an Old Version**

```bash
git checkout <commit-id> -- index.html
```

If you want to reset **all** files:

```bash
git checkout <commit-id> -- *
```

Return to the latest version:

```bash
git checkout master -- *
```

## **6ï¸âƒ£ Undoing Mistakes in Git**

### **ğŸ”¹ Unstage a File (Before Commit)**

```bash
git restore --staged index.html
```

This moves the file back to the working directory.

### **ğŸ”¹ Undo Last Commit (Keep Changes)**

```bash
git reset --soft HEAD~1
```

### **ğŸ”¹ Undo Last Commit (Discard Changes)**

```bash
git reset --hard HEAD~1
```

## **8ï¸âƒ£ Branching & Merging**

### **ğŸ”¹ Step 14: Create & Switch Branches**

```bash

git checkout new-feature
```

Or, create and switch in one command:


### **ğŸ”¹ Step 15: Make Changes & Merge Branch**

Modify `index.html`, then:

```bash

```

Check history with:

```bash

```

## **9ï¸âƒ£ Handling Merge Conflicts**

### **ğŸ”¹ Step 16: Create a Conflict**

1ï¸âƒ£ Open `index.html` on the `main` branch and change a line.  
2ï¸âƒ£ Switch to `new-feature` and modify the same line.  
3ï¸âƒ£ Try merging:

